Document Object Model (DOM)

@patrickhamann #perfshop
DOM is parsed and constructed incrementally, as bytes arrive
HTML5 first spec to define DOM parsing.

CSS Object Model (CSSOM)

@patrickhamann #perfshop
Although DOM construction is complete can’t render without CSS as would cause a FOUC
CSS parsing is not incremental, due to its cascading nature. Have to wait for entire file.
Potentially splitting up your CSS into multiple files may be valid idea. (mobile)

JavaScript

@patrickhamann #perfshop
  • Javascript can alter both the DOM and CSSOM
    • DOM construction cannot proceed until JS is fetched
    • DOM construction cannot proceed until JS is executed
  • JS can query CSSOM
    • JS can block on CSS
    • JS can modify CSSOM
  • i.e CSS at the top, JS at the bottom. *this is why*

Throttle and debounce all the things


function debounce(fn, delay) {
    var timer = null;
    return function () {
        var context = this, args = arguments;
        clearTimeout(timer);
        timer = setTimeout(function () {
            fn.apply(context, args);
        }, delay);
    };
}
                    

$('window').on('scroll', debounce(function (event) {
    console.log('tick');
}, 1000));
@patrickhamann #perfshop
Debounce waits for the end of the event chain
Throttle limits to every n milliseconds
